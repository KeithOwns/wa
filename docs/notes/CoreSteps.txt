  The steps in the core.ps1 script coincide with wa.ps1 in two different ways: steps that are both
  audited and remediated/configured by  wa.ps1 , and steps that are only audited (checked by the scanner but not set by
  the configuration engine).

  Here is the breakdown:

  ### 1. Steps that are both Audited and Configured/Applied by  wa.ps1

  These steps in  core.ps1  have matching detection checks and active remediation scripts in  wa.ps1  to configure
  them:

  • Real-Time Protection:
      •  core.ps1 :  Set-MpPreference -DisableRealtimeMonitoring $false
      •  wa.ps1 : Audited in scanner via  $mp.DisableRealtimeMonitoring  and applied/remediated by wa.ps1.
  • PUA App Blocking:
      •  core.ps1 :  Set-MpPreference -PUAProtection 1
      •  wa.ps1 : Audited in scanner via  $mp.PUAProtection  and applied/remediated by wa.ps1.
  • Memory Integrity (HVCI):
      •  core.ps1 : Sets  HypervisorEnforcedCodeIntegrity\Enabled  to  1 .
      •  wa.ps1 : Audited in scanner via registry check and applied/remediated by wa.ps1.
  • Windows Firewall:
      •  core.ps1 :  Set-NetFirewallProfile -Enabled True
      •  wa.ps1 : Audited in scanner via  Get-NetFirewallProfile  and applied/remediated by wa.ps1.
  • Microsoft Product Updates:
      •  core.ps1 : Sets  AllowMUUpdateService  to  1 .
      •  wa.ps1 : Audited in scanner via registry check and applied/remediated by wa.ps1.

  ──────
  ### 2. Steps that are Audited Only by  wa.ps1  (Checked by the scanner but not configured/applied)

  These security hardening settings are checked and reported in the completion/audit report generated by
  wa.ps1, but  wa.ps1  does not contain any active functions/buttons to configure or remediate them:

  • PowerShell Transcription:
      •  core.ps1 : Sets  EnableTranscripting  to  1 .
      •  wa.ps1 : Checked at line wa.ps1 of the posture audit block.
  • Disable Telemetry:
      •  core.ps1 : Sets  AllowTelemetry  to  0 .
      •  wa.ps1 : Checked at line wa.ps1 of the posture audit block.
  • Disable LLMNR:
      •  core.ps1 : Sets  EnableMulticast  to  0 .
      •  wa.ps1 : Checked at line wa.ps1 of the posture audit block.

The AtomicScripts (WA) engine has been fully refactored for high-contrast visibility and expanded remediation. 

Key Architectural Logic:
1. ALWAYS RUN STEPS:
   - Admin Check, Execution Policy, File Unblocking.
   - UTF-8 Encoding & Window Snapping.
   - Global Discovery Scans (populates UI and SmartRUN).
   - System Pre-Flight (OS info, Uptime, Space).
   - Post-Flight Posture Audit (AtomicScripts_Audit.json export).

2. CONDITIONAL REMEDIATION:
   - Controlled by $Global:Toggle variables.
   - SmartRUN: Audits state first, skips if compliant.
   - Manual: Forces execution of all toggled items.

3. UI THEME:
   - Palette: Cyan (Action/Selection) and Greyscale (Status/Brackets).
   - Margin-based cursors (index 0 and 55).
   - Tri-state toggle for NetBIOS (0: None, 1: Secure, 2: Default).
